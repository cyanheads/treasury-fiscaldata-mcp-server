# 0.1.4 — 2026-06-12

## Added

- **`totalCount`** output field on `treasury_query_dataset` — the full number of rows matching the query across all pages, surfaced via `ctx.enrich.total()` so the agent knows the returned page is a subset.
- **Docker `HEALTHCHECK`** — bun-native `fetch` against `/healthz` (the slim runtime image ships no `curl`/`wget`).
- **`org.opencontainers.image.version`** OCI label, populated from the `APP_VERSION` build arg.

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.19 → ^0.10.6`.
- **Display identity** — `createApp()` now sets `name` and `title` explicitly to `treasury-fiscaldata-mcp-server`; `.codex-plugin` `interface.displayName` de-scoped from `@cyanheads/treasury-fiscaldata-mcp-server` to the bare repo name.
- **SQL gate** — the framework's `denySystemCatalogs: true` now owns denial of `information_schema`, `pg_catalog`, `sqlite_master`, `duckdb_*`, and the `pragma_*` namespace; `sql-gate-extras` is reduced to the one residual the framework misses, `which_secret()`.
- **`.mcpbignore`** patterns root-anchored (`/skills/` etc.) so they match only top-level paths; `bundle` now runs `scripts/clean-mcpb.ts` after `mcpb pack` to strip dependency-shipped agent-doc trees the pack step cannot exclude.
- **`check-framework-antipatterns`** gains a `z.coerce.boolean()` rule — boolean env flags can't be disabled through the environment (`Boolean("false") === true`); use `z.stringbool()`.
