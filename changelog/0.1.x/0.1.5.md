# 0.1.5 — 2026-06-20

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.10.6 → ^0.10.9`. Brings the `ctx.content` collector for image/audio blocks ([cyanheads/mcp-ts-core#239](https://github.com/cyanheads/mcp-ts-core/issues/239)), Canvas SQL-gate `invalid_sql` classification preserving DuckDB binder detail ([cyanheads/mcp-ts-core#236](https://github.com/cyanheads/mcp-ts-core/issues/236)), the `DuckdbProvider.describe({ tableName })` ambiguous-column fix ([cyanheads/mcp-ts-core#235](https://github.com/cyanheads/mcp-ts-core/issues/235)), and corrected `OTEL_SERVICE_NAME` seeding docs ([cyanheads/mcp-ts-core#233](https://github.com/cyanheads/mcp-ts-core/issues/233)).
- **`devcheck` gains two static guards** (re-synced from the framework scripts):
  - **`check-dependency-specifiers`** — new `scripts/check-dependency-specifiers.ts`, registered as the `Dependency Specifiers` step (`--no-dep-specifiers`); hard-fails on floating specifiers (`latest`, and `*`/dist-tags in `dependencies`/`devDependencies`) in `package.json` and `bun.lock`'s `workspaces` map. Catches `bun update --latest` writing a literal `latest` dist-tag into the lock that re-resolves past an intentional hold. ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
  - **`lint:packaging` check 10** — validates plugin marketplace manifests (`.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, `.codex-plugin/mcp.json`): non-empty descriptions, unscoped machine name in display fields, full `package.json` name in the install arg. Gated by the new `devcheck.config.json` `packaging.pluginManifests` flag (on). ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))
- **`.codex-plugin/plugin.json` `interface.longDescription`** populated (was empty) to satisfy the new packaging gate.
- **Fresh-scaffold devcheck robustness** — `build-changelog`, `devcheck`, `check-framework-antipatterns`, and `check-skill-versions` re-synced with the framework's git-absent / worktree-deletion guards. ([cyanheads/mcp-ts-core#237](https://github.com/cyanheads/mcp-ts-core/issues/237), [cyanheads/mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [cyanheads/mcp-ts-core#243](https://github.com/cyanheads/mcp-ts-core/issues/243))
- **Agent skills re-synced** to the framework's 0.10.7–0.10.9 versions — `api-context` documents `ctx.content`; `api-auth`, `api-errors`, `api-services`, `api-telemetry`, `field-test`, `report-issue-local`, `tool-defs-analysis` pick up body drift; `git-wrapup` and `orchestrations` to v1.4. ([cyanheads/mcp-ts-core#238](https://github.com/cyanheads/mcp-ts-core/issues/238))
