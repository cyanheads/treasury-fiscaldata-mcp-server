# 0.1.1 — 2026-06-01

Initial public release of `@cyanheads/treasury-fiscaldata-mcp-server`. Provides 7 MCP tools over the [US Treasury Fiscal Data API](https://fiscaldata.treasury.gov/api-documentation/):

- **`treasury_list_datasets`** — browse the embedded catalog of 80+ Treasury endpoints by category or keyword
- **`treasury_query_dataset`** — query any endpoint by path with field selection, filter conditions, sort, and pagination; optional DataCanvas spill for large result sets
- **`treasury_get_debt`** — national debt (Debt to the Penny) for latest, specific date, or date-range series
- **`treasury_get_interest_rates`** — average Treasury interest rates by security type (monthly, since 2001)
- **`treasury_get_exchange_rates`** — official statutory exchange rates for ~130 countries (quarterly)
- **`treasury_dataframe_describe`** — list registered DataCanvas dataframes with schema, row count, and TTL
- **`treasury_dataframe_query`** — run single-statement SELECT SQL against DataCanvas dataframes via DuckDB

## Added

- **`treasury_list_datasets`**, **`treasury_query_dataset`**, **`treasury_get_debt`**, **`treasury_get_interest_rates`**, **`treasury_get_exchange_rates`**, **`treasury_dataframe_describe`**, **`treasury_dataframe_query`** — full 7-tool surface over Treasury Fiscal Data API
- **`maybeRegisterDataframe`** helper in `canvas-bridge.ts` — consolidates DataCanvas spill logic shared across tools
- **`sql-gate-extras.ts`** — server-level SQL deny list for DuckDB pragma table-valued functions not covered by the framework's built-in gate (`pragma_database_size`, `which_secret`)
- **`bunfig.toml`** — Bun config for consistent installs across environments
- **DataCanvas support** (`CANVAS_PROVIDER_TYPE=duckdb`) — tools can spill large result sets to named DuckDB tables for follow-up SQL queries

## Changed

- **`Dockerfile`** — copies `node_modules` from the build stage so `@duckdb/node-api`'s pre-compiled native binary is preserved in the production image without reinstalling build tools
- **`.claude-plugin/plugin.json`**, **`.codex-plugin/mcp.json`** — `CANVAS_PROVIDER_TYPE=duckdb` added to default env; install args pinned to `@latest`
- **`CLAUDE.md`** / **`AGENTS.md`** — replaced scaffold placeholder sections with live server overview (tool table, services, DataCanvas note, real structure tree)
- **`.env.example`** — documents `CANVAS_PROVIDER_TYPE` and `CANVAS_TTL_MS` DataCanvas env vars

## Security

- **`sql-gate-extras.ts`** — blocks `pragma_database_size()` and `which_secret()` DuckDB pragma TVFs; both expose internal metadata and bypass the framework's existing function deny list because they lower into generic plan operators
